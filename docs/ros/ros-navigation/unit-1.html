1 Navigation Stack

  • set of ros nodes and algorithms used to make robot move autonomously
  • takes input of
    • current location
    • desired location
    • odom data(wheel encoders,IMU,GPS)
    • laser scan (optional)
  • outputs velocuty commands cmd_vel
  • the nav stack is generic - can work with any moving robots
  • works better with diff-drive and holonomic
  • better with square and circular dhaped bots

explanation of the blocks in figure  1

  • Odometery source - current w.r.t to starting position. Main sources are wheel encoders,IMU,2d/3d cameras (visual odom) - [msg type : nav_msg/Odometery]
  • sensor source - localizing (laser scans) & detecting obstacles (sonar,pcl)
  • sensor transforms - each sensor must be refrenced with robots common frame(base_link)
  • base_controller-converts nav stack msg to motor velocities

2 the move_base

  • most imp node in nav stack
  • moves the robot from current position to goal position with help of other nav nodes
